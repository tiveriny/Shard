# Security Policy

This is an experimental project. Not audited. Do not use for critical security needs.

## Supported Versions

Only the current code in the main published branch is expected to receive fixes. No long-term support policy is promised.

## Reporting a Vulnerability

- Prefer a private maintainer contact or a forge security advisory if your hosting platform supports it.
- If no private channel is available, do not publish working exploit details in a public issue first.
- Share the affected browser and server versions, impact, reproduction steps, and whether prior device compromise is required.

## Response Expectations

- Triage is best effort.
- Acknowledge reports as quickly as possible.
- Coordinate disclosure after a fix or mitigation is available.

## Current Security Limits

- No independent audit has been completed.
- A compromised browser, device, server, or reverse proxy can expose secrets.
- Shared hosting environments must be reviewed separately for isolation and logging risks.
- This repository does not provide a formal security guarantee.

## Scope Notes

The project is intended for experimentation, review, and contribution. Treat it as a prototype until an external audit, broader testing, and operational hardening are completed.
